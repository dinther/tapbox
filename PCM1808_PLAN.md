# PCM1808 Line-In — Wiring & Bring-Up Plan

Staged plan for adding the PCM1808 24-bit stereo line-in ADC alongside the
INMP441 mic. Each phase ends with an oscilloscope (or observable) checkpoint
that must pass before the next wire goes on — so a failure always has a
one-phase search space. Phases 1–4 touch only previously unused pins; the
device stays fully functional throughout. Phase 5 is the only rewiring phase.

Board pinout confirmation: `component_images/WT32-ETH01-pinout.png` (silk
`RXD`=IO5, `TXD`=IO17, `CFG`=IO32, `485_EN`=IO33; IO35/36/39 input-only).
PCM1808 breakout: `component_images/PCM1808.png` (silk `PMT` = datasheet FMT).

## Clocking design (settled — don't revisit without cause)

- PCM1808 runs in **Master mode**: it derives BCK/LRCK internally from SCKI,
  so there is exactly one clock domain. (Slave mode was rejected: ESP32 would
  generate SCKI via LEDC and BCK/LRCK via I2S from two independent dividers
  that drift apart → periodic PCM1808 resync mutes, datasheet §7.3.3.)
- **SCKI = 8.000 MHz** from LEDC: 80 MHz APB ÷ (divider 5 × 2¹) — integer
  divider, zero cycle-to-cycle jitter. 16 MHz is NOT achievable jitter-free
  (needs fractional divider 2.5; LEDC fractional dividers alternate two
  periods and SCKI jitter degrades the ADC, datasheet §10.1.6).
- **256fs** master mode → fs = 8 MHz ÷ 256 = **31.25 kHz** (mic stays 32 kHz;
  hop rate 61.04 Hz vs mic's 62.5 Hz — close enough that the DSP keeps its
  character, but the rate must be a per-source runtime value, not the
  compile-time constant).
- ESP32 receives as **I2S slave RX** on a second port — slave pins route via
  the GPIO matrix, any GPIO works. (The GPIO0/1/3-only restriction applies
  only to the I2S peripheral's dedicated MCLK *output*; irrelevant here.)
- Boot behaviour is inherently safe: until LEDC starts, SCKI is static, which
  the PCM1808 defines as power-down/reset (datasheet §7.4.2).

## Final wiring reference

| Signal | GPIO | Silk | Notes |
|---|---|---|---|
| PCM1808 SCKI | IO5 | `RXD` | LEDC 8 MHz output |
| PCM1808 BCK | IO17 | `TXD` | PCM1808 output → ESP32 input, 2.000 MHz |
| PCM1808 LRC | IO32 | `CFG` | PCM1808 output → ESP32 input, 31.25 kHz |
| PCM1808 OUT | IO33 | `485_EN` | audio data → ESP32 input |
| PCM1808 MD0 | — | | strap **HIGH** (10 kΩ to 3.3 V) |
| PCM1808 MD1 | — | | strap **HIGH** (10 kΩ to 3.3 V) → Master, 256fs |
| PCM1808 FMT (`PMT`) | — | | strap **LOW** (10 kΩ to GND) → I²S 24-bit |
| PCM1808 +5V / 3.3V / GND | — | | both supplies required (5 V analog, 3.3 V digital) |
| Tap + Select buttons | IO35 | | Phase 5: resistor ladder (10 kΩ pullup; tap 10 kΩ, select 22 kΩ to GND) |
| 3.5 mm jack detect | IO39 | | Phase 5: 10 kΩ pullup, jack switch contact to GND |
| INMP441 / display | IO4,12,36 / IO14,2,15 | | unchanged |

## Phase 1 — SCKI clock only (no new wiring)

- [x] Firmware: LEDC 8 MHz output on IO5 (`line_clock_start()` in main.cpp)
- [x] Flash, then scope IO5 at the header.

**Checkpoint: PASSED 2026-07-14** — 8 MHz square wave on IO5, ~3.6 Vpp on the
scope (edge overshoot; logic level is 3.3 V). Flashed as part of v1.12.1.

## Phase 2 — PCM1808 powered, master mode proven (no data pins yet)

- [x] Wire: +5V, 3.3V, GND(s), MD0/MD1/FMT straps, SCKI ← IO5.
- [x] Scope the PCM1808 board's own BCK and LRC pins.

**Checkpoint: PASSED 2026-07-14** — BCK = 2.000 MHz, LRCK = 31.25 kHz, both
generated by the PCM1808 (Master/256fs confirmed). As-built notes: breakout
has NO onboard LDO (both 5 V and 3.3 V wired externally); MD0/MD1 tied
directly to the 3.3 V rail (no strap resistors — datasheet-sanctioned);
FMT is silk-labelled `PMT` on this board. Bring-up hiccup for the record:
with SCKI unconnected the chip sits in clock-halt power-down and BCK/LRC
float (~fs-rate crosstalk pickup) — connecting SCKI woke it instantly,
no power cycle needed.

## Phase 3 — data into the ESP32

- [x] Wire: BCK → IO17, LRC → IO32, OUT → IO33.
- [x] Firmware: second I2S port as slave RX (Philips format, 24-in-32-bit
      slots, stereo), debug serial line printing frame level in dB.
- [x] Test: phone playing a sine tone into LIN/RIN via 3.5 mm.

**Checkpoint: PASSED 2026-07-14** — idle floor ~-85 dB both channels;
440 Hz sine from phone → -22.3/-21.8 dB L/R, rock steady, ~15 744 frames
per 500 ms window (≈31.25 kHz confirmed end-to-end). Frames flowing also
settled IO17 = silk `TXD` empirically (vendor pinout image listing IO5
twice was a typo). No ESP32 slave-RX alignment quirks observed. 3.5 mm
socket: tip→LIN, ring→RIN, sleeve→analog GND pin between LIN/RIN.

## Phase 4 — into the beat pipeline

Design changed from the original "mic/line sub-toggle" to a flat 4-source
Source setting (user request 2026-07-14): **CDJ / Mic / Tap / Line** — enum
values append-only (`MODE_LINE = 3`) so persisted settings survive upgrades;
"Manual" renamed to "Tap" everywhere ("tAP" on-device). Mic and Line share
all audio-mode behavior (tap anchor, range gate, lock/PLL) via
`mode_is_audio()`.

- [x] Sample rate per-source at runtime (31250 line-in / 32000 mic); mel
      filterbank re-setup on source switch (top band 15 kHz on line-in,
      under its 15.625 kHz Nyquist).
- [x] mic_task reads whichever I2S channel the Source selects (single
      reader; Phase 3's line_task removed). Line-in stereo is averaged to
      mono into the same ring buffer. Full pipeline reset on switch.
- [x] BPM frame-rate correction: BTrack hard-codes kFrameHz = 62.5, so
      line-in readings are scaled by 61.035/62.5 (×0.97656) at the consumer
      in mic_task — without this every line-in BPM would read 2.4 % high.
- [x] UI: web Source dropdown lists CDJ / Mic / Line / Tap; device menu
      cycles CdJ → Aud → tAP → LinE ("Mic" cannot render on 7-seg — no M —
      so the mic source keeps its Aud glyph). Normal-mode source bar:
      CDJ=top, tAP=middle, Mic=bottom, Line=top+bottom ("≡").
- [x] **Checkpoint: PASSED 2026-07-14** — "rock solid high confidence
      tracking" through line-in (v1.13.0). Only Phase 5 (button ladder +
      jack detect) remains.

## Phase 5 — button ladder + jack detect (the only rewiring phase, last)

- [ ] Rewire: buttons to resistor ladder on IO35 (levels: idle 3.3 V /
      select 2.27 V / tap 1.65 V / both 1.34 V); IO39 → jack switch contact.
- [ ] Firmware: ADC1 decode of 4 ladder levels (20 ms debounce window,
      N-consistent-reads style); jack-detect auto-switch to line-in.
- [ ] Flash and rewire together — old button wiring stops working the moment
      the ladder firmware runs.

**Checkpoint:** all button functions work (tap, select, both-held combos for
OTA/reset); plugging a 3.5 mm cable auto-switches source to line-in.
